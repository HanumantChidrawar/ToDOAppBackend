Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listened on the user's end to identify all available Normal(non-admin) users that are currently online.
The output will be an object, object has key as userId and value as full name.
auth-error event
This event ("auth-error") has to be listened to know if any error has occurred on socket.
This event is emitted to every user of Meeting Planner Application.
userId event
This event ("userId") has to be listened to get the notifications about received / accepted / rejected friend request and unfriend action.
Every user has to listen on its own userId for the friend/ Item/ subItem/ List updates.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token.
The following data has to be emitted:
authentication token
eg- ("set-user",authToken);
updateFriend event
This event ("updateFriend") has to be emitted when you received/rejected/accept a friend request or unFriend someone.
This event will notify your friend about the action taken by You. The following data has to be emitted:
eg- data = { friendId:"Id of the user who is to be notified about request", toast:"Message which is to be shown to user" };
("updateFriend",data);
listUpdate event
This event ("listUpdate") has to be emitted when you create/update/delete a ToDo List.
This event will notify your friend about the action taken by You. The following data has to be emitted:
eg- data = { friends:"Array of Friends Id's which are to be notified about todo list", toast:"Message which is to be shown to user" };
("listUpdate",data);
itemUpdate event
This event ("itemUpdate") has to be emitted when you create/update/delete item/subItem of ToDo list.
This event will notify your friend about the action taken by You. The following data has to be emitted:
eg- data = { users:"Array of users Id's of todo lists whoose item/subItems are updated which are to be notified about todo list", toast:"Message which is to be shown to user" };
("itemUpdate",data);
disconnection
This event ("disconnection") has to be emitted, when the user gets disconnected(logout).
eg- ("disconnect","");
